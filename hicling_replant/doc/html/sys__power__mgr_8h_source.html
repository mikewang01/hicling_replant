260  * \brief Put the system to idle or sleep or blocks in a WFI() waiting for the next tick, if neither
261  * idle nor sleep is possible.
262  *
263  * \detail Puts the system to idle or sleep, if possible. If an exit-from-idle or a wake-up is
264  * needed, it programs the Timer1 to generate an interrupt after the specified idle or sleep
265  * period. Else, the system stays forever in idle or sleep mode.
266  * If neither idle nor sleep is possible, it blocks in a WFI() call waiting for the next
267  * (already programmed) OS tick to hit.
268  *
269  * \param[in] xLowPowerPeriods The number of (prescaled) low power clock periods the OS will be
270  * idle. If it is 0 then the OS indicates that it can block forever waiting for an
271  * external event. If the system goes to sleep, then it can wake up only from an external
272  * event in this case.

274  * \return void
